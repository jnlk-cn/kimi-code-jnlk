# 技能设计中的说服原则

## 概述

LLM 与人类一样会对相同的说服原则产生反应。理解这种心理学有助于你设计更有效的技能——不是为了操纵，而是为了确保关键实践即使在压力下也能被遵循。

**研究基础：** Meincke 等人（2025）对 N=28,000 次 AI 对话测试了 7 种说服原则。说服技巧使遵从率提升了一倍以上（33% → 72%，p < .001）。

## 七大原则

### 1. 权威性（Authority）
**定义：** 对专业知识、资质或官方来源的服从。

**在技能中的应用方式：**
- 命令式语言："YOU MUST"、"Never"、"Always"
- 不可协商的框架："No exceptions"
- 消除决策疲劳和自我合理化

**使用场景：**
- 强制纪律类技能（TDD、验证要求）
- 安全关键实践
- 已确立的最佳实践

**示例：**
```markdown
✅ 先写代码再写测试？删掉它。重新开始。没有例外。
❌ 在可行的情况下，可以考虑先写测试。
```

### 2. 承诺一致性（Commitment）
**定义：** 与先前行为、声明或公开承诺保持一致。

**在技能中的应用方式：**
- 要求主动声明："Announce skill usage"
- 强制明确选择："Choose A, B, or C"
- 使用跟踪：待办清单用于检查项

**使用场景：**
- 确保技能真正被遵循
- 多步骤流程
- 问责机制

**示例：**
```markdown
✅ 当你找到一个技能时，必须声明："I'm using [Skill Name]"
❌ 可以考虑告诉你的搭档你正在使用哪个技能。
```

### 3. 稀缺性（Scarcity）
**定义：** 来自时间限制或有限可用性的紧迫感。

**在技能中的应用方式：**
- 时间约束要求："Before proceeding"
- 顺序依赖："Immediately after X"
- 防止拖延

**使用场景：**
- 即时验证要求
- 时间敏感的工作流
- 防止"我稍后再做"

**示例：**
```markdown
✅ 完成任务后，必须立即请求代码审查，然后才能继续。
❌ 你可以在方便的时候审查代码。
```

### 4. 社会认同（Social Proof）
**定义：** 从众于他人行为或公认规范。

**在技能中的应用方式：**
- 普遍模式："Every time"、"Always"
- 失败模式："X without Y = failure"
- 建立规范

**使用场景：**
- 记录通用实践
- 警告常见失败
- 强化标准

**示例：**
```markdown
✅ 没有 todo 跟踪的检查清单 = 步骤会被跳过。每次都是如此。
❌ 有些人觉得待办清单对检查项有帮助。
```

### 5. 共同体（Unity）
**定义：** 共享身份、"我们感"、群体归属感。

**在技能中的应用方式：**
- 协作式语言："our codebase"、"we're colleagues"
- 共同目标："we both want quality"

**使用场景：**
- 协作工作流
- 建立团队文化
- 非层级实践

**示例：**
```markdown
✅ 我们是共同合作的同事。我需要你诚实的技术判断。
❌ 如果我错了，你大概应该告诉我。
```

### 6. 互惠性（Reciprocity）
**定义：** 对获得的好处产生回报的义务。

**应用方式：**
- 谨慎使用——可能让人感觉被操纵
- 在技能中很少需要

**避免场景：**
- 几乎总是避免（其他原则更有效）

### 7. 喜好性（Liking）
**定义：** 更愿意与我们喜欢的人合作。

**应用方式：**
- **不要用于获取遵从**
- 与诚实的反馈文化相冲突
- 会产生谄媚附和

**避免场景：**
- 在纪律执行中永远避免

## 按技能类型组合原则

| 技能类型 | 推荐使用 | 避免使用 |
|------------|-----|-------|
| 强制纪律类 | Authority + Commitment + Social Proof | Liking, Reciprocity |
| 指导/技术类 | 适度 Authority + Unity | 过度权威 |
| 协作类 | Unity + Commitment | Authority, Liking |
| 参考类 | 仅清晰度 | 所有说服技巧 |

## 为什么有效：心理学原理

**清晰边界规则减少合理化：**
- "YOU MUST" 消除了决策疲劳
- 绝对化语言消除了"这是否是例外？"的疑问
- 明确的反合理化表述堵住了特定漏洞

**执行意图创造自动行为：**
- 清晰触发条件 + 必要行动 = 自动执行
- "When X, do Y" 比 "generally do Y" 更有效
- 降低遵从的认知负担

**LLM 是类人的：**
- 在人类文本上训练，其中包含这些模式
- 权威语言在训练数据中先于遵从出现
- 承诺序列（声明 → 行动）被频繁建模
- 社会认同模式（每个人都做 X）建立规范

## 道德使用

**正当用途：**
- 确保关键实践被遵循
- 创建有效的文档
- 防止可预见的失败

**不当用途：**
- 为个人利益操纵
- 制造虚假紧迫感
- 基于内疚的遵从

**检验标准：** 如果用户完全理解这种技巧，它是否仍然服务于用户的真实利益？

## 研究引用

**Cialdini, R. B. (2021).** *Influence: The Psychology of Persuasion (New and Expanded).* Harper Business.
- 七大说服原则
- 影响力研究的实证基础

**Meincke, L., Shapiro, D., Duckworth, A. L., Mollick, E., Mollick, L., & Cialdini, R. (2025).** Call Me A Jerk: Persuading AI to Comply with Objectionable Requests. University of Pennsylvania.
- 对 N=28,000 次 LLM 对话测试了 7 种原则
- 使用说服技巧后遵从率从 33% 提升至 72%
- Authority、Commitment、Scarcity 最有效
- 验证了 LLM 行为的类人类模型

## 快速参考

设计技能时，问自己：

1. **它是什么类型？**（强制纪律 vs. 指导 vs. 参考）
2. **我想改变什么行为？**
3. **哪些原则适用？**（强制纪律通常用 authority + commitment）
4. **是否组合了太多原则？**（不要同时使用全部七种）
5. **这是否合乎道德？**（是否服务于用户的真实利益？）
