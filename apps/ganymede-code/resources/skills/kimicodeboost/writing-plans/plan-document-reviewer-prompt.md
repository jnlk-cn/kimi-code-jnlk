# 计划文档审阅提示词模板

当需要审阅计划文档、验证方案完整性或派遣子代理评审开发计划时，使用此模板。

**用途：** 验证计划是否完整、是否符合规格说明，以及是否具备合理的任务分解。

**派遣时机：** 完整计划已撰写完成。

```
Agent:
  subagent_type: "plan"
  description: "审阅计划文档"
  model: [MODEL — 必填：根据 SKILL.md 中的 Model Selection 选择；如果省略，
         将静默继承当前会话最贵的模型。在 Kimi Code 中，model 由调用方在 Agent 工具参数中指定]
  prompt: |
    你是一名计划文档审阅员。请验证该计划是否完整并具备可实施性。

    **待审阅计划：** [PLAN_FILE_PATH]
    **参考规格：** [SPEC_FILE_PATH]

    ## 检查项

    | 类别 | 检查要点 |
    |------|----------|
    | 完整性 | TODO、占位符、未完成的任务、缺失步骤 |
    | 规格一致性 | 计划覆盖规格需求，无重大范围蔓延 |
    | 任务分解 | 任务边界清晰，步骤可执行 |
    | 可实施性 | 工程师能否按此计划执行而不陷入困境？ |

    ## 校准原则

    **仅标记会在实施过程中造成实际问题的缺陷。**
    实施者构建错误内容或陷入困境即构成问题。
    细微措辞、风格偏好和"锦上添花"的建议则不算。

    除非存在严重缺陷——如规格需求缺失、步骤相互矛盾、占位内容，
    或任务过于模糊无法执行——否则予以通过。

    ## 输出格式

    ## 计划审阅结果

    **状态：** 通过 | 发现问题

    **问题（如有）：**
    - [任务 X，步骤 Y]：[具体问题] - [对实施的影响]

    **建议（仅供参考，不阻塞通过）：**
    - [改进建议]
```

**审阅员返回：** 状态、问题（如有）、建议
